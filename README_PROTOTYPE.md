# LOTT 模块需求文档写作规范 (README_PROTOTYPE)

**目的**: 统一 LOTT 项目各模块 README.md 的编写格式，避免层级间功能介绍重复，确保文档简洁、聚焦、可维护。

---

## 一、文档结构 (必须按顺序)

1. 需求概述
2. 技术框架
3. 已实现功能
4. 待实现功能 (PENDING)
5. 核心类定义
6. 接口说明
7. 用例示例

---

## 二、章节写作规范

### 1. 概述

- **用途**: 自然语言说明本模块在整个项目中的定位和核心价值
- **职责**: 列出本模块负责的重点业务功能，但不含技术细节
- **示例**:
  
  ```
  本模块负责从交易所接收实时行情数据，并将其标准化后写入 TimescaleDB。
  向上层模块提供统一的数据查询接口。
  ```

### 2. 技术框架

- 列出本模块使用的核心技术和依赖
- 使用 bullet point
- **示例**:
  
  ```
  - 核心框架: vn.py 2.1.3
  - 数据库: TimescaleDB (时序数据存储)
  - 通信协议: WebSocket (实时行情)
  - 外部API: CTP API (六交所接口)
  ```

### 3. 已实现功能

- 用自然语言描述已完成的功能
- 每条功能独立成行，使用 `- [x]` 标记
- **禁止**: 在此处写代码、配置示例、API 参数细节
- **示例**:
  
  ```
  - [x] 支持连接 CTP 仿真环境接收行情
  - [x] 实现了自动重连机制
  - [x] 数据标准化写入 TimescaleDB
  - [x] 提供 RESTful 查询接口
  ```

### 4. 待实现功能 (PENDING)

- 用自然语言描述尚未完成或需要修改的功能
- 每条独立成行，使用 `- [ ]` 标记
- 标注优先级: `P0` (必须) / `P1` (重要) / `P2` (可选)
- **示例**:
  
  ```
  - [ ] P0 支持连接实盘交易环境 (需配置真实CTP账户)
  - [ ] P1 添加数据缓存层减少数据库IO
  - [ ] P2 支持多策略同时订阅不同合约
  ```

### 5. 核心类定义

- **仅列出类名和简要职责**，不使用代码块展示实现细节
- 使用伪代码风格定义接口（见第6节）
- **示例**:
  
  ```
  | 类名 | 职责 |
  |------|------|
  | CtpMarketGateway | CTP行情网关，负责交易所连接 |
  | DataNormalizer | 数据标准化器，统一样式 |
  | TimescaleWriter | 时序数据库写入器 |
  | MarketDataAPI | 对外提供的数据查询接口 |
  ```

### 6. 接口说明

#### 6.1 接口定义规范

使用简洁的**伪代码**格式，每个接口包含：

- 接口名称
- 调用方向（谁调用谁）
- 输入参数（名称 + 类型 + 说明）
- 输出参数
- 异常处理

```
接口: subscribe(symbol: str, exchange: str) -> bool
调用方向: 外部模块 -> 本模块
说明: 订阅指定合约的行情数据

输入:
  - symbol: str      # 合约代码, 如 "IF2406"
  - exchange: str    # 交易所代码, 如 "CFFEX"
输出:
  - bool              # 订阅成功返回 True
异常:
  - ConnectionError   # 网关未连接
  - SymbolNotFound   # 合约代码不存在
```

#### 6.2 接口分类

| 分类        | 说明              |
| --------- | --------------- |
| **供外部调用** | 其他模块调用本模块的接口    |
| **调用外部**  | 本模块调用其他模块/系统的接口 |

- **禁止**: 在接口说明中出现大段代码实现
- **允许**: 使用 3-5 行以内的示例代码说明调用方式

#### 6.3 接口列表模板

**供外部调用 (In)**

| 接口名            | 输入                 | 输出    | 说明    |
| -------------- | ------------------ | ----- | ----- |
| subscribe      | (symbol, exchange) | bool  | 订阅行情  |
| unsubscribe    | (symbol, exchange) | bool  | 取消订阅  |
| get_last_price | (symbol)           | float | 获取最新价 |

**调用外部 (Out)**

| 接口名             | 目标模块        | 输入             | 输出   | 说明     |
| --------------- | ----------- | -------------- | ---- | ------ |
| write_db        | TimescaleDB | (symbol, data) | bool | 写入行情数据 |
| notify_strategy | Strategy模块  | (event)        | None | 推送行情事件 |

### 7. 用例示例

- 列出 2-3 个典型使用场景
- 使用自然语言描述，不写完整代码
- **示例**:

**用例1: 策略订阅行情**

1. 策略模块调用 subscribe("IF2406", "CFFEX")
2. CtpMarketGateway 建立与CTP服务器的连接
3. 收到行情后经 DataNormalizer 标准化
4. 写入 TimescaleDB 并触发 notify_strategy 事件

**用例2: 查询历史数据**

1. 外部调用 get_bars("IF2406", "2024-01-01", "2024-01-31")
2. MarketDataAPI 从 TimescaleDB 查询并返回

## 三、约束与禁止

| 规则       | 说明                   |
| -------- | -------------------- |
| 禁止大段代码   | 接口说明中的示例代码不超过 5 行    |
| 禁止复制粘贴实现 | 详细实现应放在子模块文档中        |
| 禁止功能重复   | 父级 README 只描述职责，不写细节 |
| 引用子模块    | 需要细节时链接到子模块的 README  |
| 保持简洁     | 每个章节不超过 100 字（接口表除外） |

---

## 四、层级约定

| 层级      | 文档位置                  | 内容粒度            |
| ------- | --------------------- | --------------- |
| LOTT根目录 | README.md             | 项目概览、整体架构、技术选型  |
| 一级模块    | [模块名]/README.md       | 本模块职责、核心接口、依赖关系 |
| 二级子模块   | [模块名]/[子模块]/README.md | 详细实现、配置、代码片段    |

**原则**: 父级不写子级细节，子级不复制父级概述

---

## 五、文件命名

- 主文档: README.md
- 规范模板: README_PROTOTYPE.md (本文档)
- 子模块示例: [模块名]/README.md
